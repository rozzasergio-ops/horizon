# Horizon

## Mission

Horizon does not recommend destinations.
It builds the journey a person needs to live.

## Core Principles

- Experiences come before destinations.
- Every trip is composed of scenes.
- Quality is more important than quantity.
- The AI must always explain why it recommends something.
- Never optimize for popularity alone.
- Authenticity has a higher weight than physical challenge.

## Code Rules

- Use TypeScript.
- Use React functional components.
- Keep components small.
- Never duplicate business logic.
- Use clean architecture.
- Prefer composition over inheritance.

## Project Structure

```
apps/
packages/
docs/
```

## Coding Style

Readable over clever.
Simple over complex.

Never invent travel data.
Always separate UI from Decision Engine.
